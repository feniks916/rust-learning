# День 31: Проект — Калькулятор размера позиции

## О проекте

Первый месяц позади! Сегодня мы собираем всё изученное в один полноценный проект: интерактивный калькулятор размера позиции с учётом риска, стоп-лосса и размера депозита.

## Формула Келли / риск-менеджмент

```
Размер позиции = (Депозит × Риск%) / (Стоп-лосс%)
```

## Код проекта

```rust
use std::io;

fn read_f64(prompt: &str) -> f64 {
    println!("{}", prompt);
    let mut input = String::new();
    io::stdin().read_line(&mut input).expect("Ошибка чтения");
    input.trim().parse().expect("Введите число")
}

fn position_size(balance: f64, risk_pct: f64, stop_pct: f64) -> f64 {
    (balance * risk_pct / 100.0) / (stop_pct / 100.0)
}

fn trade_levels(entry: f64, stop_pct: f64, target_pct: f64) -> (f64, f64) {
    let stop   = entry * (1.0 - stop_pct / 100.0);
    let target = entry * (1.0 + target_pct / 100.0);
    (stop, target)
}

fn main() {
    println!("╔══════════════════════════════════╗");
    println!("║  Калькулятор размера позиции     ║");
    println!("╚══════════════════════════════════╝");

    let balance    = read_f64("Депозит ($):");
    let risk_pct   = read_f64("Риск на сделку (%):");
    let entry      = read_f64("Цена входа ($):");
    let stop_pct   = read_f64("Стоп-лосс (%):");
    let target_pct = read_f64("Тейк-профит (%):");

    let size = position_size(balance, risk_pct, stop_pct);
    let (stop_price, target_price) = trade_levels(entry, stop_pct, target_pct);
    let qty = size / entry;
    let risk_amount  = balance * risk_pct / 100.0;
    let reward = qty * (target_price - entry);
    let rr_ratio = reward / risk_amount;

    println!("\n━━━━━━━━ Результат ━━━━━━━━");
    println!("Размер позиции: ${:.2}", size);
    println!("Количество:     {:.6}", qty);
    println!("Стоп-лосс:      ${:.2}", stop_price);
    println!("Тейк-профит:    ${:.2}", target_price);
    println!("Риск:           ${:.2}", risk_amount);
    println!("Потенциал:      ${:.2}", reward);
    println!("R/R ratio:      {:.2}", rr_ratio);
}
```

Полезно смотреть на проект не как на один большой листинг, а как на набор маленьких задач.

## Что делает `read_f64`

```rust
fn read_f64(prompt: &str) -> f64 {
    println!("{}", prompt);
    let mut input = String::new();
    io::stdin().read_line(&mut input).expect("Ошибка чтения");
    input.trim().parse().expect("Введите число")
}
```

По шагам:

1. выводим подсказку пользователю
2. создаём строку для ввода
3. читаем строку из терминала
4. убираем пробелы и перенос строки
5. парсим число

Это маленький, но очень полезный шаблон для консольных программ.

## Что считает `position_size`

Функция:

```rust
fn position_size(balance: f64, risk_pct: f64, stop_pct: f64) -> f64 {
    (balance * risk_pct / 100.0) / (stop_pct / 100.0)
}
```

считает денежный размер позиции по такой логике:

1. сначала находим, сколько денег мы готовы потерять
2. потом делим эту сумму на размер стопа в процентах

Пример:

1. депозит = `$10_000`
2. риск = `2%`, значит допустимый убыток = `$200`
3. стоп = `4%`
4. позиция = `$200 / 0.04 = $5000`

То есть если позиция размером `$5000` упадёт на `4%`, убыток как раз составит `$200`.

## Пошаговый разбор остальных вычислений

После размера позиции программа считает ещё несколько полезных величин:

1. `qty = size / entry` — сколько единиц актива можно купить
2. `stop_price` — цена, при которой сделка закрывается по стопу
3. `target_price` — целевая цена по тейк-профиту
4. `risk_amount` — сумма риска в деньгах
5. `reward` — потенциальная прибыль до достижения цели
6. `rr_ratio` — отношение прибыль/риск

Это уже настоящий трейдерский workflow, а не просто упражнение на деление чисел.

## Более дружелюбный ввод

В проектной программе лучше не падать на первой же опечатке пользователя. Ниже более устойчивый вариант `read_f64`:

```rust
fn read_f64(prompt: &str) -> f64 {
    loop {
        println!("{}", prompt);

        let mut input = String::new();
        io::stdin().read_line(&mut input).expect("Ошибка чтения");

        match input.trim().parse::<f64>() {
            Ok(value) if value > 0.0 => return value,
            Ok(_) => println!("Введите число больше нуля."),
            Err(_) => println!("Это не похоже на число, попробуй ещё раз."),
        }
    }
}
```

Такой вариант особенно полезен для реального проекта или домашней работы.

## Практическое расширение: предупреждение по R/R

Можно сразу сделать программу умнее и добавить проверку отношения риск/прибыль.

```rust
fn rr_warning(rr_ratio: f64) -> &'static str {
    if rr_ratio < 1.5 {
        "Внимание: слабое соотношение риск/прибыль"
    } else if rr_ratio < 2.0 {
        "Нормально, но без запаса"
    } else {
        "Хорошее соотношение риск/прибыль"
    }
}

fn main() {
    let rr_ratio = 1.42;
    println!("{}", rr_warning(rr_ratio));
}
```

Это хороший пример того, как простая математика превращается в принятие торгового решения.
