# День 37: Ссылки — смотрим на чужой портфель

## Аналогия из трейдинга

Аналитик может посмотреть на портфель клиента, не забирая активы. Ссылка в Rust — это именно такой взгляд: доступ к данным без передачи владения.

## Зачем вообще нужны ссылки

В прошлых главах мы разобрали владение и `move`. Без ссылок любое чтение больших данных быстро превратилось бы в проблему:

1. либо функция забирает данные себе, и вызывающий код больше не может ими пользоваться
2. либо приходится постоянно делать `.clone()`, что дорого и часто вообще не нужно

Ссылка решает обе проблемы. Мы говорим функции: «вот данные, посмотри на них, но не забирай».

Это особенно важно в трейдинге, где одна и та же структура может использоваться сразу в нескольких вычислениях:

1. модуль риска считает exposure
2. модуль аналитики считает доходность
3. модуль логирования печатает снимок состояния

Все они могут читать один и тот же набор данных через ссылки.

## Ссылки: &T

```rust
fn print_portfolio(portfolio: &Vec<String>) {
    // Смотрим на портфель, не владеем им
    for asset in portfolio {
        println!("  {}", asset);
    }
}

fn main() {
    let my_portfolio = vec![
        String::from("BTC"),
        String::from("ETH"),
        String::from("SOL"),
    ];

    print_portfolio(&my_portfolio); // передаём ссылку
    println!("Портфель всё ещё мой: {} активов", my_portfolio.len());
}
```

## Ссылка не передаёт владение

```rust
fn total_value(prices: &Vec<f64>) -> f64 {
    prices.iter().sum()
}

fn main() {
    let prices = vec![50000.0, 3000.0, 150.0];
    let total = total_value(&prices); // ссылка, не Move
    println!("Сумма: ${:.2}", total);
    println!("prices всё ещё тут: {:?}", prices); // ОК!
}
```

## Разыменование: *

```rust
fn add_fee(price: &f64, fee: f64) -> f64 {
    *price + fee // * разыменовывает ссылку
}

fn main() {
    let price = 50000.0;
    let with_fee = add_fee(&price, 50.0);
    println!("С комиссией: ${}", with_fee);
}
```

## Ссылки на срезы

```rust
fn max_price(prices: &[f64]) -> f64 {
    prices.iter().cloned().fold(f64::NEG_INFINITY, f64::max)
}

fn main() {
    let week_prices = vec![48000.0, 51000.0, 49500.0, 52000.0, 50000.0];
    println!("Максимум недели: ${}", max_price(&week_prices));
}
```

## Как читать тип `&T`

Новичкам полезно читать тип справа налево:

1. `Vec<String>` — вектор строк
2. `&Vec<String>` — ссылка на вектор строк
3. `&[f64]` — ссылка на срез чисел `f64`

То есть `&` почти всегда значит: «это не само значение, а доступ к нему без владения».

```rust
fn show_balance(balance: &f64) {
    println!("Баланс: {}", balance);
}

fn show_assets(assets: &Vec<String>) {
    println!("Активов: {}", assets.len());
}

fn show_window(prices: &[f64]) {
    println!("Окно содержит {} цен", prices.len());
}
```

## Что именно происходит при `&variable`

Когда ты пишешь `&my_portfolio`, Rust не копирует весь вектор. Он создаёт ссылку на уже существующие данные.

```rust
fn main() {
    let prices = vec![48_000.0, 49_000.0, 50_000.0];

    let ref1 = &prices;
    let ref2 = &prices;

    println!("Через ref1: {:?}", ref1);
    println!("Через ref2: {:?}", ref2);
    println!("Через владельца: {:?}", prices);
}
```

У нас не появилось три вектора. Вектор по-прежнему один, а способов безопасно посмотреть на него стало несколько.

## Когда нужен `*`

Разыменование `*` нужно, когда мы хотим добраться от ссылки до самого значения.

Для многих операций Rust делает это автоматически, поэтому `*` встречается не всегда. Но если тебе нужен сам `f64` в вычислении, разыменование уже полезно.

```rust
fn spread(bid: &f64, ask: &f64) -> f64 {
    *ask - *bid
}

fn main() {
    let bid = 50_000.0;
    let ask = 50_010.0;

    println!("Spread: {:.2}", spread(&bid, &ask));
}
```

## `&Vec<T>` против `&[T]`

В учебных примерах часто начинают с `&Vec<T>`, потому что это проще визуально. Но в реальном Rust-коде функция чаще принимает срез `&[T]`, потому что это более общий интерфейс.

```rust
fn average(prices: &[f64]) -> f64 {
    let sum: f64 = prices.iter().sum();
    sum / prices.len() as f64
}

fn main() {
    let v = vec![1.0, 2.0, 3.0];
    let a = [4.0, 5.0, 6.0];

    println!("Среднее Vec: {}", average(&v));
    println!("Среднее массива: {}", average(&a));
}
```

Практическое правило простое:

1. если нужен именно вектор как структура, используй `&Vec<T>`
2. если нужен просто набор элементов подряд, лучше `&[T]`

## Практический пример: отчёт по watchlist

```rust
fn print_watchlist(symbols: &[String]) {
    println!("--- Watchlist ---");
    for (index, symbol) in symbols.iter().enumerate() {
        println!("{}. {}", index + 1, symbol);
    }
}

fn longest_symbol(symbols: &[String]) -> &String {
    let mut best = &symbols[0];

    for symbol in symbols {
        if symbol.len() > best.len() {
            best = symbol;
        }
    }

    best
}

fn main() {
    let watchlist = vec![
        String::from("BTCUSDT"),
        String::from("ETHUSDT"),
        String::from("SOLUSDT"),
        String::from("ARBUSDT"),
    ];

    print_watchlist(&watchlist);

    let best = longest_symbol(&watchlist);
    println!("Самый длинный тикер: {}", best);

    println!("Исходный watchlist всё ещё доступен: {} инструментов", watchlist.len());
}
```

## Частые ошибки

### Ошибка 1: передаём владение, когда нужно только чтение

Если функция только печатает или считает статистику, чаще всего ей не нужен `String` или `Vec<T>` по владению.

### Ошибка 2: думаем, что ссылка делает копию

Ссылка не клонирует данные. Она просто смотрит на уже существующее значение.

### Ошибка 3: боимся `&[T]`

Срез — это не другая коллекция, а просто более общий способ сослаться на последовательность элементов.

## Что мы узнали

| Синтаксис | Описание |
|-----------|----------|
| `&T` | Неизменяемая ссылка на T |
| `&variable` | Создаём ссылку |
| `*reference` | Разыменовываем ссылку |
| `&[T]` | Ссылка на срез |
