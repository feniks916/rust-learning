# День 23: break — выходим когда достигли тейк-профита

## Аналогия из трейдинга

Представь: ты поставил тейк-профит на уровне $55 000. Цена растёт тик за тиком, и как только она касается цели — ты немедленно закрываешь позицию и фиксируешь прибыль. Ты не ждёшь следующего тика, не проверяешь дополнительные условия — просто выходишь. Именно так работает `break` в Rust: как только условие выполнено, выполнение цикла немедленно прерывается и программа продолжается со строки после цикла. Это один из самых важных инструментов управления потоком, и в трейдинговых алгоритмах он встречается буквально повсюду.

## Базовый break: выход из цикла по условию

Самый простой случай — цикл `for` по массиву цен, из которого мы выходим, когда цена достигла нужного уровня:

```rust
fn main() {
    let prices = [49500.0, 50100.0, 51200.0, 52800.0, 54100.0, 55300.0, 56000.0];
    let take_profit = 55000.0;

    println!("Мониторим цену BTC/USDT...");

    for (tick, price) in prices.iter().enumerate() {
        println!("Тик {}: ${:.2}", tick + 1, price);

        if *price >= take_profit {
            println!("Тейк-профит сработал на тике {}! Цена: ${:.2}", tick + 1, price);
            println!("Закрываем позицию и фиксируем прибыль.");
            break;
        }
    }

    println!("Мониторинг завершён.");
}
```

После `break` Rust немедленно переходит к строке `println!("Мониторинг завершён.")` — оставшиеся тики не обрабатываются. Это важно для производительности: зачем проверять 1000 свечей, если нужный сигнал найден на 10-й?

## break в цикле while: ожидание пробоя уровня

`while` с `break` часто используют когда заранее не знаешь, сколько итераций понадобится:

```rust
fn main() {
    let resistance = 52000.0;
    let mut price = 50000.0;
    let mut ticks = 0;

    println!("Ждём пробоя уровня сопротивления ${:.0}...", resistance);

    while price < resistance * 1.1 {
        // Симулируем движение цены
        price += price * 0.005; // +0.5% каждый тик
        ticks += 1;

        println!("Тик {}: цена ${:.2}", ticks, price);

        if price > resistance {
            println!("ПРОБОЙ! Цена ${:.2} пробила уровень ${:.0}", price, resistance);
            println!("Открываем лонг-позицию по прорыву.");
            break;
        }

        if ticks >= 20 {
            println!("Таймаут: пробоя не произошло за {} тиков", ticks);
            break;
        }
    }

    println!("Итого ожидали {} тиков.", ticks);
}
```

Здесь два `break`: один — при успешном пробое, другой — при таймауте. Это типичный паттерн: "жди события, но не вечно".

## break в бесконечном цикле loop

`loop` в Rust — специальная конструкция для бесконечного цикла, из которого выходят только через `break`. Это идеально для торгового бота, который работает непрерывно:

```rust
fn main() {
    let mut price = 50000.0;
    let stop_loss = 48000.0;
    let take_profit = 54000.0;
    let mut iteration = 0;

    println!("Торговый бот запущен. SL: ${:.0}, TP: ${:.0}", stop_loss, take_profit);

    loop {
        iteration += 1;
        // Симулируем изменение цены
        let change = if iteration % 3 == 0 { -500.0 } else { 300.0 };
        price += change;

        println!("Итерация {}: цена ${:.2}", iteration, price);

        if price <= stop_loss {
            println!("СТОП-ЛОСС! Цена упала до ${:.2}. Убыток зафиксирован.", price);
            break;
        }

        if price >= take_profit {
            println!("ТЕЙК-ПРОФИТ! Цена выросла до ${:.2}. Прибыль зафиксирована.", price);
            break;
        }
    }

    println!("Бот остановлен после {} итераций.", iteration);
}
```

`loop` без явного условия — это честно: мы говорим "работай, пока не скажу стоп". Условия выхода полностью под нашим контролем.

## break с возвратом значения из loop

Уникальная возможность Rust — вернуть значение из цикла через `break`. Это позволяет использовать `loop` как выражение:

```rust
fn find_entry_price(prices: &[f64], min_price: f64) -> Option<f64> {
    let result = loop {
        // Ищем первую цену выше минимума
        for &price in prices {
            if price > min_price {
                break Some(price); // возвращаем значение из loop
            }
        }
        break None; // если не нашли
    };
    result
}

fn calculate_first_profit(trades: &[(f64, f64)]) -> Option<f64> {
    // (цена_входа, цена_выхода)
    let profit = loop {
        for &(entry, exit) in trades {
            let pnl = exit - entry;
            if pnl > 0.0 {
                break Some(pnl);
            }
        }
        break None;
    };
    profit
}

fn main() {
    let prices = vec![48000.0, 49500.0, 51000.0, 52500.0];
    match find_entry_price(&prices, 50000.0) {
        Some(p) => println!("Первая подходящая цена входа: ${:.2}", p),
        None    => println!("Подходящих цен не найдено"),
    }

    let trades = vec![(50000.0, 49000.0), (49000.0, 48000.0), (48500.0, 51000.0)];
    match calculate_first_profit(&trades) {
        Some(p) => println!("Первая прибыльная сделка: +${:.2}", p),
        None    => println!("Прибыльных сделок нет"),
    }
}
```

`break Some(price)` — это не просто выход, это передача данных наружу. Очень элегантный паттерн для поиска первого совпадающего значения.

## Вложенные циклы и метки 'outer

Когда циклы вложены друг в друга, простой `break` выходит только из внутреннего. Для выхода из внешнего нужны метки:

```rust
fn main() {
    let days = ["Пн", "Вт", "Ср", "Чт", "Пт"];
    let sessions = ["Лондон", "Нью-Йорк", "Азия"];
    let mut trades_found = 0;
    let max_trades = 3;

    println!("Сканируем торговые сессии по дням...");

    'day_loop: for day in &days {
        for session in &sessions {
            // Симулируем нахождение торговых сигналов
            let signal = format!("{}-{}", day, session).len() % 3;

            if signal == 0 {
                trades_found += 1;
                println!("Сигнал найден: {} сессия {}, всего сигналов: {}", day, session, trades_found);

                if trades_found >= max_trades {
                    println!("Достигнут лимит сигналов {}. Прекращаем сканирование.", max_trades);
                    break 'day_loop; // выходим из ВНЕШНЕГО цикла
                }
            } else {
                println!("  {} {}: сигналов нет", day, session);
            }
        }
    }

    println!("Сканирование завершено. Найдено сигналов: {}", trades_found);
}
```

Метка `'day_loop:` ставится перед внешним циклом. `break 'day_loop` выходит из него, минуя все внутренние. Правило именования меток в Rust: всегда начинаются с апострофа `'`.

## Метки с while и двойное вложение

Метки работают с любыми циклами, не только с `for`:

```rust
fn main() {
    let weeks = 3;
    let days_per_week = 5;
    let hours_per_day = 8;
    let mut total_hours = 0;

    let emergency_day = (2, 3); // неделя 2, день 3

    'week: for week in 1..=weeks {
        'day: for day in 1..=days_per_week {
            let mut hour = 9;
            while hour < 9 + hours_per_day {
                if (week, day) == emergency_day && hour == 11 {
                    println!("Экстренная остановка торгов! Неделя {}, день {}, час {}", week, day, hour);
                    break 'week; // выходим из ВСЕХ циклов
                }
                total_hours += 1;
                hour += 1;
            }
            println!("День {}/{}: завершён", week, day);
        }
    }

    println!("Итого часов торговли до остановки: {}", total_hours);
}
```

`break 'week` прыгает через все три уровня вложенности сразу. Это гораздо чище, чем цепочка флагов `should_stop`.

## Практический пример: поиск первого пробоя уровня

Соберём всё вместе — реальный алгоритм поиска ценового пробоя в массиве свечей:

```rust
struct Candle {
    open:   f64,
    high:   f64,
    low:    f64,
    close:  f64,
    volume: f64,
}

fn find_breakout(candles: &[Candle], resistance: f64, min_volume: f64) -> Option<(usize, f64)> {
    let found = loop {
        for (i, candle) in candles.iter().enumerate() {
            // Пробой должен быть подтверждён закрытием выше уровня
            if candle.close > resistance && candle.volume >= min_volume {
                break Some((i, candle.close));
            }
            // Ложный пробой — high выше, но close ниже
            if candle.high > resistance && candle.close <= resistance {
                println!("Свеча {}: ложный пробой (high {:.2}, close {:.2})", i + 1, candle.high, candle.close);
            }
        }
        break None;
    };
    found
}

fn main() {
    let resistance = 52000.0;
    let min_volume = 1000.0;

    let candles = vec![
        Candle { open: 50000.0, high: 51500.0, low: 49800.0, close: 51200.0, volume: 850.0 },
        Candle { open: 51200.0, high: 52300.0, low: 51000.0, close: 51800.0, volume: 920.0 },
        Candle { open: 51800.0, high: 53000.0, low: 51500.0, close: 51900.0, volume: 780.0 }, // ложный
        Candle { open: 51900.0, high: 53500.0, low: 51700.0, close: 52800.0, volume: 1200.0 }, // пробой!
        Candle { open: 52800.0, high: 54000.0, low: 52500.0, close: 53500.0, volume: 1500.0 },
    ];

    println!("Ищем пробой уровня сопротивления ${:.0}...", resistance);
    println!("Минимальный объём для подтверждения: {:.0}", min_volume);
    println!();

    match find_breakout(&candles, resistance, min_volume) {
        Some((idx, price)) => {
            println!();
            println!("ПРОБОЙ ПОДТВЕРЖДЁН!");
            println!("Свеча: {}", idx + 1);
            println!("Цена закрытия: ${:.2}", price);
            println!("Сигнал: открываем лонг выше ${:.2}", resistance);
        }
        None => {
            println!("Пробоя уровня ${:.0} не найдено", resistance);
        }
    }
}
```

Здесь `break Some((i, candle.close))` возвращает сразу два значения — индекс свечи и цену. Это идиоматичный Rust: `loop` как функция поиска с ранним выходом.

## Что мы узнали

| Синтаксис | Когда использовать | Пример |
|-----------|-------------------|--------|
| `break` | Немедленный выход из текущего цикла при выполнении условия | Сработал тейк-профит → выходим из цикла мониторинга |
| `break` в `loop` | Бесконечный цикл с явным условием остановки | Торговый бот работает до SL или TP |
| `break value` | Вернуть результат поиска из `loop` | `let price = loop { ... break Some(p); }` |
| `'label: for` | Пометить внешний цикл для управления из внутреннего | Сканирование дней и сессий |
| `break 'label` | Выйти из конкретного именованного цикла | Выход из всей вложенной структуры при лимите |
| Двойной `break` | Обработка нескольких условий выхода | Пробой или таймаут |
