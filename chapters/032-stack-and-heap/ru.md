# День 32: Стек и куча — наличные и банковский счёт

## Аналогия из трейдинга

Представь, что ты трейдер. У тебя есть наличные в кошельке — ты можешь моментально расплатиться на бирже, достать купюру и убрать её. Это **стек**. Но у тебя также есть банковский счёт, где хранятся крупные суммы — доступ чуть медленнее, зато туда помещается сколько угодно денег. Это **куча**. Rust решает сам, где хранить каждый тип данных, и освобождает память автоматически — никаких утечек.

---

## Что такое стек (Stack)

Стек — это область памяти, работающая по принципу **LIFO** (Last In, First Out — последним пришёл, первым ушёл). Представь стопку торговых ордеров: ты кладёшь новый ордер сверху, и первым обрабатываешь именно его.

Ключевые свойства стека:
- Размер данных **известен на этапе компиляции**
- Выделение и освобождение **очень быстрое** (просто сдвиг указателя)
- Данные хранятся **последовательно** в памяти
- Когда функция завершается, её стековый фрейм **автоматически уничтожается**

```rust
fn calculate_pnl(entry_price: f64, exit_price: f64, qty: u32) -> f64 {
    // Все эти переменные живут на стеке
    let price_diff: f64 = exit_price - entry_price;  // 8 байт
    let quantity: f64 = qty as f64;                   // 8 байт (преобразование)
    let pnl: f64 = price_diff * quantity;             // 8 байт
    let commission: f64 = 0.001 * exit_price * quantity; // 8 байт
    let net_pnl: f64 = pnl - commission;              // 8 байт

    net_pnl
    // Когда функция вернёт значение — price_diff, quantity, pnl,
    // commission автоматически уничтожатся
}

fn main() {
    let entry: f64 = 42000.0;  // На стеке
    let exit: f64 = 45000.0;   // На стеке
    let qty: u32 = 2;           // На стеке

    let result = calculate_pnl(entry, exit, qty);
    println!("Чистый PnL: ${:.2}", result);
}
```

Числовые типы (`i8`, `i16`, `i32`, `i64`, `u8`, `u16`, `u32`, `u64`, `f32`, `f64`, `bool`, `char`) всегда живут на стеке — их размер фиксирован.

## Что такое куча (Heap)

Куча — это большая область памяти для данных, **размер которых неизвестен заранее или меняется во времени**. Как банковский счёт: ты можешь положить сколько угодно, но каждая операция требует чуть больше усилий.

Особенности кучи:
- Размер данных может быть **любым** и изменяться в runtime
- Выделение памяти через аллокатор (чуть медленнее)
- Данные **не обязательно рядом** в памяти
- Освобождение в Rust происходит **автоматически через систему владения** (никакого `free()` вручную)

```rust
fn main() {
    // String живёт на куче — мы не знаем заранее, сколько символов
    let ticker = String::from("BTCUSDT");

    // Vec<f64> живёт на куче — список цен может расти
    let mut prices: Vec<f64> = Vec::new();
    prices.push(42000.0);
    prices.push(42500.0);
    prices.push(41800.0);
    prices.push(43000.0);

    println!("Тикер: {}", ticker);
    println!("Последняя цена: ${}", prices.last().unwrap());
    println!("Всего свечей: {}", prices.len());
    // Когда main() завершится — ticker и prices автоматически
    // освобождаются, память возвращается операционной системе
}
```

## Какие типы где живут

Правило простое: если компилятор знает размер типа — стек, если нет — куча.

```rust
fn main() {
    // === СТЕК ===
    let price: f64 = 50000.0;        // 8 байт — стек
    let volume: u64 = 1_000_000;     // 8 байт — стек
    let is_long: bool = true;         // 1 байт — стек
    let side: char = 'B';             // 4 байта — стек
    let order_id: i32 = 12345;        // 4 байта — стек
    let coords: (f64, f64) = (1.0, 2.0); // 16 байт — стек (кортеж известного размера)
    let last_three: [f64; 3] = [100.0, 101.0, 99.0]; // 24 байта — стек (массив фикс. размера)

    // === КУЧА ===
    let symbol = String::from("ETHUSDT");     // данные символа на куче
    let history: Vec<f64> = vec![1.0, 2.0];  // данные вектора на куче
    let nested: Vec<Vec<f64>> = vec![         // и внешний, и внутренние — на куче
        vec![1.0, 2.0],
        vec![3.0, 4.0],
    ];

    println!("Цена: {}, Символ: {}", price, symbol);
    println!("История: {:?}", history);
    drop(nested); // явно освобождаем, хотя Rust сделал бы это сам
}
```

## Box\<T\> — помещаем стековые данные в кучу

`Box<T>` — умный указатель, который принудительно размещает данные в куче. Это как положить наличные (стековые данные) в банк (кучу).

Зачем это нужно:
- Для рекурсивных типов (дерево, связный список), где размер типа зависит от самого типа
- Для передачи больших структур без копирования
- Для работы с типажами через `dyn Trait`

```rust
// Без Box — рекурсивный тип не скомпилируется!
// enum OrderBook { Node(f64, OrderBook) } // ОШИБКА: бесконечный размер

// С Box — компилятор знает размер Box (просто указатель)
#[derive(Debug)]
enum PriceLevel {
    Empty,
    Level {
        price: f64,
        volume: f64,
        next: Box<PriceLevel>, // Box имеет фиксированный размер (указатель)
    },
}

fn main() {
    let order_book = PriceLevel::Level {
        price: 50000.0,
        volume: 1.5,
        next: Box::new(PriceLevel::Level {
            price: 49999.0,
            volume: 3.0,
            next: Box::new(PriceLevel::Empty),
        }),
    };

    // Просто большая структура в куче через Box
    let big_price_array: Box<[f64; 1000]> = Box::new([0.0; 1000]);
    println!("Размер Box: {} байт", std::mem::size_of_val(&big_price_array));
    // Box сам по себе — маленький (8 байт указатель), данные в куче
}
```

## Практическое значение для Rust

Понимание стека и кучи объясняет многие решения языка:

```rust
fn main() {
    // i32 копируется при присваивании (Copy trait)
    // Потому что копировать 4 байта со стека — мгновенно
    let price1: i32 = 42000;
    let price2 = price1;       // копия, не перемещение
    println!("price1={}, price2={}", price1, price2); // ОБА работают

    // String перемещается при присваивании (Move semantics)
    // Потому что копировать данные с кучи — дорого (нужно выделить новую память)
    // Rust по умолчанию ПЕРЕМЕЩАЕТ, а не копирует
    let ticker1 = String::from("BTCUSDT");
    let ticker2 = ticker1;     // перемещение владения
    // println!("{}", ticker1); // ОШИБКА: ticker1 больше не владеет данными
    println!("ticker2={}", ticker2); // только ticker2 действителен

    // Чтобы скопировать String — нужно явно вызвать .clone()
    let s1 = String::from("ETHUSDT");
    let s2 = s1.clone();       // явная дорогая копия
    println!("s1={}, s2={}", s1, s2); // оба работают
}
```

## Диаграмма памяти в коде

Посмотрим, как именно данные располагаются в памяти:

```rust
fn show_memory_layout() {
    // Стековый фрейм функции show_memory_layout:
    // [  price: f64  ][  qty: u32  ][  ticker: (ptr|len|cap)  ]
    //  0x7fff0000      0x7fff0008    0x7fff000C
    //      ↓                              ↓
    //   50000.0           10         указатель на кучу
    //                                     ↓
    //                              Куча: ['B','T','C','U','S','D','T']

    let price: f64 = 50000.0;
    let qty: u32 = 10;
    let ticker = String::from("BTCUSDT");

    // String на стеке хранит ТРИ вещи:
    // 1. ptr — указатель на данные в куче
    // 2. len — текущая длина (7)
    // 3. cap — ёмкость (обычно >= len)
    println!("Размер самой строки-структуры на стеке: {} байт",
             std::mem::size_of::<String>()); // 24 байта (3 × 8)
    println!("Размер данных в куче: {} байт", ticker.len()); // 7 байт

    // Vec аналогично:
    let prices: Vec<f64> = vec![1.0, 2.0, 3.0];
    println!("Размер Vec-структуры на стеке: {} байт",
             std::mem::size_of::<Vec<f64>>()); // 24 байта
    println!("Цена: {}, Кол-во: {}", price, qty);
}

fn main() {
    show_memory_layout();
}
```

## Практический пример: Анализатор торговых данных

Соберём всё вместе — создадим структуру, которая правильно использует стек и кучу:

```rust
#[derive(Debug)]
struct TradeSession {
    // Поля фиксированного размера — будут на стеке внутри структуры
    session_id: u32,
    open_price: f64,
    close_price: f64,
    is_profitable: bool,

    // Поля динамического размера — данные в куче, указатели на стеке
    symbol: String,
    trade_prices: Vec<f64>,
    notes: String,
}

impl TradeSession {
    fn new(symbol: &str, open_price: f64) -> TradeSession {
        TradeSession {
            session_id: 1,
            open_price,
            close_price: 0.0,
            is_profitable: false,
            symbol: String::from(symbol),  // выделяет память в куче
            trade_prices: Vec::new(),       // пустой вектор в куче
            notes: String::new(),
        }
    }

    fn add_trade(&mut self, price: f64) {
        self.trade_prices.push(price);  // может расширить кучу
        self.close_price = price;
        self.is_profitable = price > self.open_price;
    }

    fn summary(&self) -> String {  // возвращает новую String (куча)
        format!(
            "Сессия {}: {} | Открытие: ${:.2} | Закрытие: ${:.2} | {} | Сделок: {}",
            self.session_id,
            self.symbol,
            self.open_price,
            self.close_price,
            if self.is_profitable { "ПРИБЫЛЬ" } else { "УБЫТОК" },
            self.trade_prices.len()
        )
    }

    fn average_price(&self) -> f64 {  // возвращает f64 (стек)
        if self.trade_prices.is_empty() {
            return self.open_price;
        }
        let sum: f64 = self.trade_prices.iter().sum();
        sum / self.trade_prices.len() as f64
    }
}

fn main() {
    // session живёт на стеке (структура), данные String/Vec — на куче
    let mut session = TradeSession::new("BTCUSDT", 42000.0);

    session.add_trade(42500.0);
    session.add_trade(43000.0);
    session.add_trade(42800.0);
    session.add_trade(44000.0);

    println!("{}", session.summary());
    println!("Средняя цена: ${:.2}", session.average_price());

    // Box<TradeSession> — вся структура в куче (для больших структур)
    let boxed_session: Box<TradeSession> = Box::new(
        TradeSession::new("ETHUSDT", 3000.0)
    );
    println!("Размер указателя Box: {} байт", std::mem::size_of_val(&boxed_session));
    // Когда main() завершается — всё освобождается автоматически:
    // boxed_session -> Drop для Box -> Drop для TradeSession -> освобождение кучи
}
```

## Что мы узнали

| Синтаксис | Описание | Пример в трейдинге |
|-----------|----------|--------------------|
| `let x: f64 = 1.0` | Переменная на стеке | Цена ордера — известный размер |
| `String::from("X")` | Строка в куче | Тикер инструмента |
| `Vec::new()` | Вектор в куче | История цен |
| `Box::new(x)` | Помещает данные в кучу | Большая структура стакана |
| `std::mem::size_of::<T>()` | Размер типа в байтах | Анализ использования памяти |
| `drop(x)` | Явное освобождение | Закрыть позицию досрочно |
| `Vec::push(v)` | Добавление в кучу | Новая сделка в истории |
