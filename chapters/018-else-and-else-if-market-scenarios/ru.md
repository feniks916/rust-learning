# День 18: else и else if — три сценария рынка

## Аналогия из трейдинга

Любой опытный трейдер всегда думает сценариями: что я делаю ЕСЛИ рынок растёт? А ЕСЛИ падает? А ЕСЛИ боковик? Хороший торговый план — это не "покупаю и молюсь", а чёткий набор ветвей: **бычий рынок** — покупаем на откатах, **медвежий рынок** — шортим на отскоках, **боковик** — торгуем от уровней или вообще отдыхаем. В Rust конструкции `else` и `else if` — это и есть такой торговый план: мы точно описываем, что делать при каждом сценарии, и ни одна ситуация не остаётся без ответа.

## else: когда условие не выполнено

`else` — это "иначе". Если `if` не сработал, выполняем блок `else`. Это бинарное решение: либо одно, либо другое.

```rust
fn main() {
    let btc_price = 48000.0;
    let target_price = 50000.0;

    if btc_price >= target_price {
        println!("Цена достигла цели — ПРОДАЁМ!");
        println!("Фиксируем прибыль по ${}", target_price);
    } else {
        println!("Цена ниже цели — ДЕРЖИМ позицию");
        println!("Ждём ${:.0}", target_price);
    }
}
```

**Важно:** `else` не имеет условия — он выполняется в любом случае, если `if` не сработал. Это гарантирует, что мы никогда не останемся без реакции на рынок.

```rust
fn main() {
    let pnl = -350.0; // текущая прибыль/убыток в USD

    let result = if pnl >= 0.0 {
        "Прибыльная позиция"
    } else {
        "Убыточная позиция"
    };

    println!("Статус: {} (${:.2})", result, pnl);
}
```

Когда использовать `else`:
- Ровно два варианта развития событий
- Бинарные состояния: открыт/закрыт, длинный/короткий, прибыль/убыток
- Значение по умолчанию если условие не выполнено

## else if: несколько сценариев RSI

Рынок редко бывает только "хорошим" или "плохим" — он многогранен. `else if` позволяет описать столько сценариев, сколько нужно.

```rust
fn classify_rsi(rsi: f64) -> &'static str {
    if rsi < 20.0 {
        "Экстремальная перепроданность — сильный сигнал BUY"
    } else if rsi < 30.0 {
        "Перепроданность — умеренный сигнал BUY"
    } else if rsi < 45.0 {
        "Слабость — осторожная покупка"
    } else if rsi < 55.0 {
        "Нейтральная зона — HOLD"
    } else if rsi < 70.0 {
        "Сила — осторожная продажа"
    } else if rsi < 80.0 {
        "Перекупленность — умеренный сигнал SELL"
    } else {
        "Экстремальная перекупленность — сильный сигнал SELL"
    }
}

fn main() {
    println!("RSI 15: {}", classify_rsi(15.0));
    println!("RSI 28: {}", classify_rsi(28.0));
    println!("RSI 50: {}", classify_rsi(50.0));
    println!("RSI 72: {}", classify_rsi(72.0));
    println!("RSI 85: {}", classify_rsi(85.0));
}
```

Ещё пример — классификация дневного изменения цены:

```rust
fn classify_price_change(change_pct: f64) -> &'static str {
    if change_pct > 10.0 {

    } else if change_pct > -10.0 {
```

## Приоритет проверки

Это КРИТИЧЕСКИ важно понимать: проверки выполняются сверху вниз, и **выполняется только первый подходящий блок**. Остальные пропускаются.

```rust
fn main() {
    let rsi = 25.0;

    // Проверки идут по порядку:
    if rsi < 30.0 {
        println!("Перепродан");   // <- выполнится ЭТО (rsi=25 < 30)
    } else if rsi < 20.0 {
        // Сюда мы НИКОГДА не попадём при rsi=25,
        // потому что уже сработал блок выше!
        println!("Экстремальная перепроданность");
    } else {
        println!("Нейтрально");
    }
}
```

**Правильный порядок — от частного к общему (от меньшего к большему):**

```rust
fn check_rsi_correct(rsi: f64) {
    // Правильно: сначала самое строгое условие
    if rsi < 20.0 {
        println!("Экстремальная перепроданность"); // rsi < 20
    } else if rsi < 30.0 {
        println!("Перепроданность");               // 20 <= rsi < 30
    } else if rsi < 70.0 {
        println!("Нейтрально");                    // 30 <= rsi < 70
    } else if rsi < 80.0 {
        println!("Перекупленность");               // 70 <= rsi < 80
    } else {
        println!("Экстремальная перекупленность"); // rsi >= 80
    }
}

fn main() {
    check_rsi_correct(15.0); // Экстремальная перепроданность
    check_rsi_correct(25.0); // Перепроданность
    check_rsi_correct(55.0); // Нейтрально
    check_rsi_correct(75.0); // Перекупленность
    check_rsi_correct(90.0); // Экстремальная перекупленность
}
```

## Вложенные if/else: анализ по цене И объёму

Иногда нужно сначала проверить один параметр, и только потом — другой. Это вложенные условия.

```rust
fn analyze_market(price: f64, sma: f64, volume: f64, avg_volume: f64) -> &'static str {
    if price > sma {
        // Цена выше SMA — бычья зона
        if volume > avg_volume * 1.5 {
            "Сильный бычий тренд (высокий объём подтверждает)"
        } else if volume > avg_volume {
            "Умеренный бычий тренд (объём в норме)"
        } else {
            "Слабый бычий сигнал (низкий объём — осторожно)"
        }
    } else {
        // Цена ниже SMA — медвежья зона
        if volume > avg_volume * 1.5 {
            "Сильный медвежий тренд (объём подтверждает продажи)"
        } else if volume > avg_volume {
            "Умеренный медвежий тренд"
        } else {
            "Слабый медвежий сигнал (низкий объём — возможен отскок)"
        }
    }
}

fn main() {
    println!("{}", analyze_market(51000.0, 49000.0, 2500.0, 1500.0));
    println!("{}", analyze_market(51000.0, 49000.0, 800.0, 1500.0));
    println!("{}", analyze_market(47000.0, 49000.0, 3000.0, 1500.0));
    println!("{}", analyze_market(47000.0, 49000.0, 900.0, 1500.0));
}
```

Важно не злоупотреблять вложенностью — больше 2-3 уровней делает код нечитаемым. В таких случаях лучше вынести логику в отдельные функции.

## if/else как выражение: возвращаем значение

В Rust `if/else` — это выражение, которое **возвращает значение**. Можно присвоить результат переменной прямо из `if`.

```rust
fn main() {
    let price_change = -3.5; // процент изменения за день

    // if/else как выражение
    let signal = if price_change > 5.0 {
        "STRONG_BUY"
    } else if price_change > 2.0 {
        "BUY"
    } else if price_change > -2.0 {
        "HOLD"
    } else if price_change > -5.0 {
        "SELL"
    } else {
        "STRONG_SELL"
    };

    println!("Изменение цены: {:.1}%", price_change);
    println!("Сигнал: {}", signal);
}
```

**Важно:** все ветви должны возвращать один и тот же тип!

```rust
fn main() {
    let balance = 15000.0;
    let trade_value = 5000.0;

    // Вычисляем комиссию: VIP если объём > $10k
    let commission_rate = if trade_value > 10000.0 {
        0.05_f64  // VIP: 0.05%
    } else {
        0.10_f64  // Standard: 0.10%
    };

    let commission = trade_value * commission_rate / 100.0;
    let after_commission = trade_value - commission;

    println!("Объём сделки: ${:.2}", trade_value);
    println!("Комиссия: {:.2}% = ${:.2}", commission_rate, commission);
    println!("Итого получаем: ${:.2}", after_commission);

    // Хватает ли баланса?
    let can_trade = if balance >= trade_value {
        "Да, баланс достаточный"
    } else {
        "Нет, недостаточно средств"
    };
    println!("Можем торговать: {}", can_trade);
}
```

## Классификация рынка: функция с 5 зонами RSI

Напишем профессиональную функцию, которая анализирует рынок по нескольким факторам одновременно:

```rust
#[derive(Debug)]
struct MarketAnalysis {
    rsi_zone: &'static str,
    trend: &'static str,
    action: &'static str,
    confidence: &'static str,
}

fn analyze_rsi(rsi: f64, price: f64, sma: f64) -> MarketAnalysis {
    let trend = if price > sma { "BULL" } else { "BEAR" };

    let (rsi_zone, action, confidence) = if rsi < 20.0 {
        ("Экстремальная перепроданность", "STRONG BUY", "Высокая")
    } else if rsi < 30.0 {
        ("Перепроданность", "BUY", "Средняя")
    } else if rsi < 45.0 {
        ("Слабость рынка", "Осторожная покупка", "Низкая")
    } else if rsi < 55.0 {
        ("Нейтральная зона", "HOLD", "Нет сигнала")
    } else if rsi < 70.0 {
        ("Сила рынка", "Осторожная продажа", "Низкая")
    } else if rsi < 80.0 {
        ("Перекупленность", "SELL", "Средняя")
    } else {
        ("Экстремальная перекупленность", "STRONG SELL", "Высокая")
    };

    MarketAnalysis { rsi_zone, trend, action, confidence }
}

fn main() {
    let analysis = analyze_rsi(25.0, 42000.0, 44000.0);
    println!("RSI зона: {}", analysis.rsi_zone);
    println!("Тренд: {}", analysis.trend);
    println!("Действие: {}", analysis.action);
    println!("Уверенность: {}", analysis.confidence);
}
```

## Практический пример: система торговых сигналов

```rust
/// Определяет торговый сигнал по комплексу индикаторов
fn get_trading_signal(
    price: f64,
    sma_20: f64,
    sma_50: f64,
    rsi: f64,
    volume: f64,
    avg_volume: f64,
) -> &'static str {
    // Определяем направление тренда по SMA
    let trend = if price > sma_20 && sma_20 > sma_50 {
        "strong_bull"
    } else if price > sma_20 {
        "weak_bull"
    } else if price < sma_20 && sma_20 < sma_50 {
        "strong_bear"
    } else {
        "weak_bear"
    };

    // Объём подтверждает движение?
    let volume_ok = volume > avg_volume;

    // Финальное решение на основе тренда + RSI + объём
    if trend == "strong_bull" {
        if rsi < 70.0 && volume_ok {
            "STRONG BUY — тренд + объём подтверждают"
        } else if rsi >= 70.0 {
            "WAIT — бычий тренд, но RSI перекуплен"
        } else {
            "BUY — бычий тренд (объём слабый)"
        }
    } else if trend == "weak_bull" {
        if rsi < 50.0 && volume_ok {
            "BUY — слабый бычий + RSI не перегрет"
        } else {
            "HOLD — неопределённость"
        }
    } else if trend == "strong_bear" {
        if rsi > 30.0 && volume_ok {
            "STRONG SELL — медвежий тренд подтверждён"
        } else if rsi <= 30.0 {
            "WAIT — медвежий тренд, но RSI перепродан"
        } else {
            "SELL — медвежий тренд (объём слабый)"
        }
    } else {
        "HOLD — нет чёткого направления"
    }
}

fn main() {
    println!("=== Система торговых сигналов ===\n");

    // Сценарий 1: Сильный бычий рынок
    let signal1 = get_trading_signal(
        52000.0, // price
        50000.0, // sma_20
        48000.0, // sma_50
        55.0,    // rsi
        2000.0,  // volume
        1500.0,  // avg_volume
    );
    println!("Сценарий 1 (бычий рынок): {}", signal1);

    // Сценарий 2: Перекупленность
    let signal2 = get_trading_signal(
        55000.0,
        50000.0,
        48000.0,
        78.0,    // RSI перекуплен
        2200.0,
        1500.0,
    );
    println!("Сценарий 2 (перекупленность): {}", signal2);

    // Сценарий 3: Медвежий рынок с подтверждением
    let signal3 = get_trading_signal(
        44000.0,
        46000.0,
        48000.0,
        42.0,
        2500.0,
        1500.0,
    );
    println!("Сценарий 3 (медвежий рынок): {}", signal3);

    // Сценарий 4: Перепроданность в медвежьем тренде
    let signal4 = get_trading_signal(
        43000.0,
        46000.0,
        48000.0,
        22.0,    // RSI перепродан — возможный отскок
        1800.0,
        1500.0,
    );
    println!("Сценарий 4 (перепроданность): {}", signal4);

    println!("\n=== Анализ завершён ===");
}
```

## Что мы узнали

| Синтаксис | Когда использовать | Пример |
|-----------|-------------------|--------|
| `if { } else { }` | Два варианта: да/нет | Прибыль или убыток |
| `else if condition { }` | 3+ сценария | Зоны RSI: 5–7 уровней |
| Порядок проверки | Сверху вниз, первое совпадение | Сначала `< 20`, потом `< 30` |
| Вложенные `if/else` | Когда условия зависят друг от друга | Цена И объём |
| `let x = if { } else { }` | if как выражение для присвоения | `let signal = if ... { "BUY" } else { "SELL" }` |
| Типы должны совпадать | Все ветви `if`-выражения | Нельзя в одной ветви `&str`, в другой `i32` |
