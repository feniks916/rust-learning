# День 28: Ввод пользователя — трейдер вводит размер позиции

## Аналогия из трейдинга

Торговый терминал принимает ввод трейдера: размер позиции, цену входа, тикер. В Rust `std::io::stdin()` — это клавиатура терминала: читаем что вводит пользователь и используем в программе.

## Чтение строки из stdin

```rust
use std::io;

fn main() {
    println!("Введите тикер (например, BTC):");

    let mut ticker = String::new();
    io::stdin()
        .read_line(&mut ticker)
        .expect("Ошибка чтения");

    let ticker = ticker.trim(); // убираем символ новой строки
    println!("Вы выбрали: {}", ticker);
}
```

## Что делает `read_line` пошагово

Новичку важно понимать, что `stdin()` не «возвращает строку», а записывает ввод в уже существующую строку.

Порядок работы такой:

1. создаём пустую строку `String::new()`
2. передаём изменяемую ссылку `&mut ticker`
3. `read_line` дописывает туда символы, которые ввёл пользователь
4. в конце строки обычно остаётся `\n`, поэтому мы делаем `.trim()`

```rust
use std::io;

fn main() {
    let mut input = String::new();

    io::stdin()
        .read_line(&mut input)
        .expect("Ошибка чтения");

    println!("Сырые данные: {:?}", input);
    println!("После trim: {:?}", input.trim());
}
```

Так проще увидеть, что пользовательский ввод и «очищенный» ввод — это не одно и то же.

## Ввод числа

```rust
use std::io;

fn main() {
    println!("Введите размер позиции ($):");

    let mut input = String::new();
    io::stdin()
        .read_line(&mut input)
        .expect("Ошибка чтения");

    let size: f64 = input.trim()
        .parse()
        .expect("Введите число");

    println!("Размер позиции: ${:.2}", size);

    let fee = size * 0.001;
    println!("Комиссия: ${:.2}", fee);
}
```

## Несколько полей ввода

```rust
use std::io;

fn read_line() -> String {
    let mut s = String::new();
    io::stdin().read_line(&mut s).expect("Ошибка");
    s.trim().to_string()
}

fn main() {
    println!("=== Калькулятор позиции ===");

    println!("Тикер:");
    let ticker = read_line();

    println!("Цена входа ($):");
    let entry: f64 = read_line().parse().expect("Число");

    println!("Количество:");
    let qty: f64 = read_line().parse().expect("Число");

    let total = entry * qty;
    println!("\n{}: {} @ ${:.2} = ${:.2}", ticker, qty, entry, total);
}
```

## Почему почти всегда нужен `trim()`

Когда пользователь нажимает Enter, в строку попадает символ новой строки. Если сразу вызвать `.parse()`, он часто помешает преобразованию.

```rust
fn main() {
    let input = "50000\n";

    // let price: f64 = input.parse().unwrap(); // часто даст ошибку
    let price: f64 = input.trim().parse().unwrap();

    println!("Цена: {}", price);
}
```

Практическое правило простое: если строка пришла от пользователя, почти всегда сначала делай `.trim()`.

## Более удобная функция чтения числа

Если программа просит несколько чисел подряд, полезно завернуть ввод в функцию с повтором до корректного результата.

```rust
use std::io;

fn read_f64(prompt: &str) -> f64 {
    loop {
        println!("{}", prompt);

        let mut input = String::new();
        io::stdin().read_line(&mut input).expect("Ошибка чтения");

        match input.trim().parse::<f64>() {
            Ok(value) => return value,
            Err(_) => println!("Нужно ввести число, попробуй ещё раз."),
        }
    }
}

fn main() {
    let risk = read_f64("Риск на сделку (%):");
    println!("Риск = {:.2}%", risk);
}
```

Такой подход лучше, чем `expect`, если программа должна быть дружелюбной к пользователю.

## Практический пример: интерактивный риск-калькулятор

```rust
use std::io;

fn read_f64(prompt: &str) -> f64 {
    loop {
        println!("{}", prompt);

        let mut input = String::new();
        io::stdin().read_line(&mut input).expect("Ошибка чтения");

        match input.trim().parse::<f64>() {
            Ok(value) if value > 0.0 => return value,
            Ok(_) => println!("Число должно быть положительным."),
            Err(_) => println!("Некорректный ввод. Введите число."),
        }
    }
}

fn main() {
    let balance = read_f64("Размер депозита ($):");
    let risk_pct = read_f64("Риск на сделку (%):");

    let risk_amount = balance * risk_pct / 100.0;

    println!("Можно рискнуть суммой: ${:.2}", risk_amount);
}
```

Здесь у пользователя уже более реальный опыт:

1. программа задаёт вопрос
2. читает ввод
3. валидирует его
4. только потом делает расчёт

## Частые ошибки

### Ошибка 1: забыли `mut` у строки для ввода

`read_line` хочет записать данные в строку, поэтому нужна изменяемая строка и `&mut String`.

### Ошибка 2: парсим без `trim()`

Очень частая причина непонятных ошибок в начале обучения.

### Ошибка 3: используем `expect` везде подряд

Для первого знакомства это нормально, но интерактивные программы лучше делать через `match` и повторный запрос.

## Что мы узнали

| Синтаксис | Описание |
|-----------|----------|
| `io::stdin().read_line(&mut s)` | Читаем строку с клавиатуры |
| `.trim()` | Убираем пробелы и `\n` |
| `.parse::<f64>()` | Конвертируем строку в число |
| `.expect("msg")` | Паника с сообщением если ошибка |
