# День 263: Риск-менеджмент — размер позиции

## Аналогия из трейдинга

Профессиональный трейдер никогда не ставит весь депозит на одну сделку. Риск-менеджмент определяет: сколько можно потерять на одной сделке, и как рассчитать размер позиции чтобы соблюдать этот лимит.

## Базовая формула

```
Размер позиции = (Депозит × Риск%) / (Стоп-лосс в $)
Стоп-лосс в $ = Цена входа × Стоп%
```

## Что означает эта формула простыми словами

Логика риск-менеджмента всегда начинается не с вопроса «сколько я хочу заработать», а с вопроса «сколько я готов потерять, если идея окажется неверной».

Порядок мышления такой:

1. выбираем допустимый риск на сделку
2. определяем, где стоит стоп-лосс
3. из этих двух чисел вычисляем допустимый размер позиции

То есть размер позиции — это не произвольное число. Он выводится из заранее заданного лимита убытка.

## Реализация

```rust
#[derive(Debug)]
struct RiskParams {
    balance: f64,
    risk_pct: f64,    // процент от баланса на риск (например 0.02 = 2%)
    stop_loss_pct: f64, // стоп-лосс в процентах от цены входа
}

#[derive(Debug)]
struct PositionSizing {
    position_value: f64,  // стоимость позиции в $
    quantity: f64,        // количество единиц
    stop_price: f64,      // цена стоп-лосса
    max_loss: f64,        // максимальный убыток в $
}

fn calculate_position(entry: f64, params: &RiskParams) -> PositionSizing {
    let max_loss     = params.balance * params.risk_pct;
    let stop_loss_$  = entry * params.stop_loss_pct;
    let position_val = max_loss / params.stop_loss_pct;
    let quantity     = position_val / entry;
    let stop_price   = entry * (1.0 - params.stop_loss_pct);

    PositionSizing {
        position_value: position_val,
        quantity,
        stop_price,
        max_loss,
    }
}

fn main() {
    let params = RiskParams {
        balance: 10_000.0,
        risk_pct: 0.02,       // 2% риска
        stop_loss_pct: 0.05,  // стоп на 5%
    };

    let entry_price = 50000.0;
    let sizing = calculate_position(entry_price, &params);

    println!("=== Расчёт размера позиции ===");
    println!("Депозит:        ${:.2}", params.balance);
    println!("Риск на сделку: {:.1}% = ${:.2}", params.risk_pct * 100.0, sizing.max_loss);
    println!("Цена входа:     ${:.2}", entry_price);
    println!("Стоп-лосс:      ${:.2} (-{:.1}%)", sizing.stop_price, params.stop_loss_pct * 100.0);
    println!("Размер позиции: ${:.2}", sizing.position_value);
    println!("Количество:     {:.6}", sizing.quantity);
}
```

## Разбираем расчёт по шагам

Предположим:

1. депозит `$10_000`
2. риск на сделку `2%`
3. цена входа `$50_000`
4. стоп `5%`

Тогда:

1. максимальный допустимый убыток = `$10_000 * 0.02 = $200`
2. стоп в деньгах на одну единицу = `$50_000 * 0.05 = $2_500`
3. размер позиции в деньгах = `$200 / 0.05 = $4_000`
4. количество актива = `$4_000 / $50_000 = 0.08`

Это и есть главное преимущество формульного подхода: размер позиции не берётся из эмоций.

## Абсолютный стоп и процентный стоп

Иногда стоп задаётся в процентах, а иногда в долларах. Это два близких, но не одинаковых сценария.

```rust
fn position_size_by_stop_amount(balance: f64, risk_pct: f64, stop_amount: f64) -> f64 {
    let max_loss = balance * risk_pct;
    max_loss / stop_amount
}
```

Если ты уже знаешь денежный риск на единицу, можно считать напрямую. Если известен только процент, сначала переводи его в денежное расстояние от цены входа.

## Почему нельзя смотреть только на размер депозита

Два трейдера с одинаковым балансом могут открыть позиции совершенно разного размера, если у них разные стопы.

Например:

1. узкий стоп 2% позволяет взять более крупную позицию
2. широкий стоп 10% требует меньшую позицию при том же лимите риска

То есть сам по себе депозит ничего не говорит без расстояния до стоп-лосса.

## Практический пример: размер позиции и целевая прибыль

```rust
fn risk_amount(balance: f64, risk_pct: f64) -> f64 {
    balance * risk_pct
}

fn reward_amount(entry: f64, target: f64, quantity: f64) -> f64 {
    (target - entry) * quantity
}

fn main() {
    let balance = 12_000.0;
    let risk_pct = 0.015;
    let entry = 2_500.0;
    let stop_pct = 0.04;
    let target = 2_750.0;

    let max_loss = risk_amount(balance, risk_pct);
    let position_value = max_loss / stop_pct;
    let quantity = position_value / entry;
    let stop_price = entry * (1.0 - stop_pct);
    let reward = reward_amount(entry, target, quantity);
    let rr = reward / max_loss;

    println!("Макс. убыток: ${:.2}", max_loss);
    println!("Размер позиции: ${:.2}", position_value);
    println!("Количество: {:.4}", quantity);
    println!("Стоп-цена: ${:.2}", stop_price);
    println!("Потенциальная прибыль: ${:.2}", reward);
    println!("R/R ratio: {:.2}", rr);
}
```

Теперь глава связывает сразу три вещи:

1. риск на сделку
2. размер позиции
3. потенциальную награду

Это уже полноценная основа для оценки сделки.

## Kelly Criterion: почему с ним осторожны

Критерий Келли даёт теоретически оптимальную долю капитала, если у тебя есть надёжные оценки вероятности выигрыша и среднего выигрыша/проигрыша. На практике проблема в том, что эти оценки редко бывают стабильными.

Поэтому многие трейдеры используют не полный Kelly, а:

1. half-Kelly
2. quarter-Kelly
3. фиксированный процент риска вроде 1-2%

То есть Kelly — полезный ориентир, но не повод мгновенно увеличивать риск.

## Частые ошибки

### Ошибка 1: рисковать фиксированной суммой без учёта стопа

Размер позиции всегда должен зависеть от расстояния до стоп-лосса.

### Ошибка 2: путать `0.02` и `2.0`

Если в коде `risk_pct = 0.02`, это уже 2%, а не 0.02%.

### Ошибка 3: считать только риск, но не смотреть на потенциальную награду

Даже идеально рассчитанная позиция может быть плохой сделкой, если `R/R` слишком слабый.

## Kelly Criterion

```rust
fn kelly_fraction(win_rate: f64, avg_win: f64, avg_loss: f64) -> f64 {
    let b = avg_win / avg_loss; // коэффициент выплат
    let p = win_rate;
    let q = 1.0 - win_rate;
    // Kelly% = (b*p - q) / b
    (b * p - q) / b
}

fn main() {
    let kelly = kelly_fraction(0.55, 200.0, 100.0);
    println!("Kelly fraction: {:.2}%", kelly * 100.0);
    println!("Рекомендуемый риск: {:.2}%", kelly * 100.0 / 2.0); // half-Kelly для безопасности
}
```

## Что мы узнали

| Формула | Описание |
|---------|----------|
| `balance × risk%` | Максимальный убыток за сделку |
| `max_loss / stop%` | Размер позиции |
| `position / entry` | Количество единиц |
| Kelly Criterion | Оптимальная доля от баланса |
