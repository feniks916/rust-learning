# День 16: Комментарии — документируем торговую логику

## Аналогия из трейдинга

Представь: ты нанял нового трейдера в команду, даёшь ему доступ к боту, а там тысячи строк кода без единого пояснения. Что делает переменная `x`? Почему умножаем на `0.00075`? Зачем вот этот `if` с магическим числом `1337`? Через месяц даже ты сам не вспомнишь, что имел в виду. В профессиональном трейдинге каждый ордер логируется с причиной — почему открыт, по какой стратегии, какой риск. Комментарии в коде — это точно такой же журнал сделок, только для программиста. Хорошо задокументированный торговый бот — это актив, плохо задокументированный — мина замедленного действия.

## Зачем нужны комментарии?

Посмотри на этот код без комментариев. Попробуй понять, что он делает:

```rust
fn main() {
    let x = 42000.0;
    let y = 41500.0;
    let z = 0.5;
    let r = 35.0;
    let v = 1800.0;
    let mv = 1500.0;

    if x < y && r < 40.0 && v > mv {
        let q = 10000.0 * 0.02 / 0.05;
        println!("GO: {}", q);
    }
}
```

Что такое `x`, `y`, `z`, `r`, `v`, `mv`? Что значит `GO`? Почему `0.02` и `0.05`? Теперь посмотри на тот же код с комментариями:

```rust
fn main() {
    let btc_price = 42000.0;       // текущая цена BTC/USDT
    let sma_20 = 41500.0;          // 20-периодная скользящая средняя
    let quantity = 0.5;            // количество BTC (не используется здесь)
    let rsi = 35.0;                // индекс относительной силы (RSI)
    let volume = 1800.0;           // объём за последний час
    let min_volume = 1500.0;       // минимальный объём для подтверждения сигнала

    // Сигнал на покупку: цена ниже SMA, RSI перепродан, объём достаточный
    if btc_price < sma_20 && rsi < 40.0 && volume > min_volume {
        // Размер позиции = депозит * риск_на_сделку / стоп_лосс_процент
        let position_size = 10000.0 * 0.02 / 0.05; // 2% риска, стоп 5%
        println!("Сигнал BUY! Размер позиции: ${}", position_size);
    }
}
```

Разница огромная! Код стал читаемым как книга.

## Однострочные комментарии //

Самый частый вид комментариев. Ставим `//` и всё до конца строки — комментарий.

**Комментарии к переменным:**

```rust
fn main() {
    let ticker = "BTCUSDT";        // торговая пара на Binance
    let entry_price = 42000.0;     // цена входа в позицию (USD)
    let stop_loss = 40000.0;       // уровень стоп-лосса (-4.76%)
    let take_profit = 46000.0;     // уровень тейк-профита (+9.52%)
    let quantity = 0.25;           // размер позиции в BTC
    let leverage = 3;              // кредитное плечо (x3)

    let risk_reward = (take_profit - entry_price) / (entry_price - stop_loss);
    println!("Тикер: {}", ticker);
    println!("Risk/Reward ratio: {:.2}", risk_reward); // должен быть >= 2.0
}
```

**Комментарии к формулам — объясняем, ПОЧЕМУ такая формула:**

```rust
fn main() {
    let deposit = 10000.0;
    let risk_percent = 0.01;       // рискуем 1% от депозита на сделку
    let stop_distance = 0.05;      // стоп на 5% от цены входа

    // Формула Kelly Criterion (упрощённая): размер = риск_в_деньгах / стоп_в_деньгах
    // Это даёт нам количество единиц актива с правильным управлением капиталом
    let risk_amount = deposit * risk_percent;  // $100 — максимальная потеря
    let position_size = risk_amount / stop_distance; // $100 / 5% = $2000

    println!("Максимальный риск: ${:.2}", risk_amount);
    println!("Размер позиции: ${:.2}", position_size);
}
```

**Комментарии к условиям:**

```rust
fn main() {
    let rsi = 28.0;
    let volume = 2200.0;
    let avg_volume = 1500.0;
    let spread = 0.15;             // спред в процентах

    // Проверяем условия для входа в позицию
    if rsi < 30.0 {
        // RSI ниже 30 = классическая перепроданность по Wilder
        println!("RSI сигнал: зона перепроданности");
    }

    if volume > avg_volume * 1.5 {
        // Объём в 1.5 раза выше среднего подтверждает импульс движения
        println!("Объём подтверждён: аномально высокий");
    }

    if spread < 0.2 {
        // Спред > 0.2% делает сделку нерентабельной при нашей стратегии
        println!("Спред приемлемый, можно торговать");
    }
}
```

**Комментарий в конце строки или на отдельной строке:**

```rust
fn main() {
    // Инициализация параметров стратегии Bollinger Bands
    let period = 20;                   // период для расчёта средней
    let std_multiplier = 2.0;          // количество стандартных отклонений
    let price = 48500.0;
    let upper_band = 50000.0;          // верхняя полоса Боллинджера
    let lower_band = 47000.0;          // нижняя полоса Боллинджера
    let middle_band = (upper_band + lower_band) / 2.0; // средняя линия

    println!("Период: {}, Множитель: {}", period, std_multiplier);
    println!("Средняя линия: {:.2}", middle_band);

    // Цена у нижней полосы — возможный сигнал покупки
    if price < lower_band * 1.01 {
        println!("Цена у нижней полосы Боллинджера — смотрим на разворот");
    }
}
```

## Многострочные комментарии /* */

Используем для больших блоков текста — например, документирование стратегии в начале файла.

**Документирование торговой стратегии:**

```rust
/*
    СТРАТЕГИЯ: RSI Reversal with Volume Confirmation
    ================================================
    Автор: AlgoTrader
    Версия: 2.1
    Дата создания: 2024-01-15

    ОПИСАНИЕ:
    Стратегия ищет точки разворота на основе индикатора RSI.
    Покупаем при перепроданности (RSI < 30), продаём при
    перекупленности (RSI > 70). Объём используется как фильтр
    ложных сигналов.

    ПАРАМЕТРЫ:
    - Таймфрейм: 1H (часовой)
    - Инструмент: BTC/USDT, ETH/USDT
    - Депозит: от $1000
    - Максимальный риск на сделку: 2%

    СТАТИСТИКА (бэктест 2020-2024):
    - Winrate: 58%
    - Profit Factor: 1.85
    - Max Drawdown: 12%
    - Sharpe Ratio: 1.42
*/

fn rsi_strategy(rsi: f64, volume: f64, avg_volume: f64) -> &'static str {
    let volume_confirmed = volume > avg_volume; // объём выше среднего

    if rsi < 30.0 && volume_confirmed {
        "BUY"   // перепроданность подтверждена объёмом
    } else if rsi > 70.0 && volume_confirmed {
        "SELL"  // перекупленность подтверждена объёмом
    } else {
        "HOLD"  // нет чёткого сигнала
    }
}
```

## Документационные комментарии ///

Это особые комментарии Rust — они генерируют HTML-документацию через команду `cargo doc`. Используй их для всех публичных функций.

```rust
/// Рассчитывает размер позиции на основе риск-менеджмента.
///
/// Использует формулу: `position = (deposit * risk_pct) / stop_pct`
/// Это гарантирует, что при срабатывании стоп-лосса мы теряем
/// не более заданного процента от депозита.
///
/// # Arguments
///
/// * `deposit` - размер торгового депозита в USD (должен быть > 0)
/// * `risk_pct` - доля депозита под риском (0.01 = 1%, типично 0.01-0.03)
/// * `stop_pct` - расстояние до стоп-лосса от цены входа (0.05 = 5%)
///
/// # Returns
///
/// Размер позиции в USD. Если входные параметры нулевые — возвращает 0.0.
///
/// # Panics
///
/// Паникует если `stop_pct` равен нулю (деление на ноль).
///
/// # Example
///
/// ```
/// let size = calculate_position_size(10000.0, 0.02, 0.05);
/// assert_eq!(size, 4000.0); // рискуем $200 при стопе 5%
/// ```
fn calculate_position_size(deposit: f64, risk_pct: f64, stop_pct: f64) -> f64 {
    if deposit <= 0.0 || risk_pct <= 0.0 {
        return 0.0;
    }
    (deposit * risk_pct) / stop_pct
}

/// Проверяет, является ли цена ордера валидной для размещения.
///
/// # Arguments
///
/// * `price` - цена ордера в USD
/// * `min_price` - минимально допустимая цена (обычно 0.01)
/// * `max_price` - максимально допустимая цена для данной пары
///
/// # Returns
///
/// `true` если цена в допустимом диапазоне, иначе `false`
///
/// # Example
///
/// ```
/// assert!(is_valid_price(42000.0, 0.01, 100000.0));
/// assert!(!is_valid_price(-100.0, 0.01, 100000.0));
/// ```
fn is_valid_price(price: f64, min_price: f64, max_price: f64) -> bool {
    price >= min_price && price <= max_price
}

fn main() {
    let pos = calculate_position_size(10000.0, 0.02, 0.05);
    println!("Размер позиции: ${:.2}", pos);

    let valid = is_valid_price(42000.0, 0.01, 200000.0);
    println!("Цена валидна: {}", valid);
}
```

## Хорошие vs плохие комментарии

Не все комментарии одинаково полезны. Плохой комментарий — это шум, который только мешает.

| Плохой комментарий | Хороший комментарий |
|--------------------|---------------------|
| `let price = 100.0; // присваиваем цену 100` | `let price = 100.0; // минимальная цена для ордера на Binance` |
| `i = i + 1; // увеличиваем i на 1` | `tick_count += 1; // считаем тики для расчёта VWAP` |
| `// TODO` | `// TODO: добавить проверку спреда перед входом (issue #42)` |
| `// умножаем на 0.001` | `// 0.001 = стандартная комиссия Binance для Maker-ордеров` |
| `// функция расчёта прибыли` | `// Используем абсолютное значение т.к. шорт даёт отрицательную разницу` |

Правило: хороший комментарий отвечает на вопрос **ПОЧЕМУ**, а не **ЧТО**. Код сам показывает ЧТО происходит — объясняй намерение и причину.

## Как НЕ надо комментировать

**Анти-паттерн 1: Комментарий повторяет код**

```rust
// ПЛОХО: комментарий бесполезен — код и так понятен
let total = price * quantity; // умножаем цену на количество

// ХОРОШО: объясняем бизнес-логику
let total = price * quantity; // стоимость позиции до учёта комиссии
```

**Анти-паттерн 2: Закомментированный мёртвый код**

```rust
// ПЛОХО: захламляет файл, никто не знает, можно ли удалить
// let old_fee = 0.002;
// let fee = amount * old_fee;
// println!("старая комиссия: {}", fee);
let fee = amount * 0.001; // актуальная комиссия с 2023 года

// ХОРОШО: удали мёртвый код, git history сохранит историю
let fee = amount * 0.001; // актуальная комиссия Binance Spot (с 2023)
```

**Анти-паттерн 3: Устаревший комментарий (хуже чем его отсутствие!)**

```rust
// ОПАСНО: комментарий лжёт! Это вызовет баг при чтении кода
// Стоп-лосс = 3% от цены входа
let stop_distance = entry_price * 0.05; // <- на самом деле 5%!

// ХОРОШО: актуализируем комментарий при изменении кода
let stop_distance = entry_price * 0.05; // стоп = 5% от цены входа
```

**Анти-паттерн 4: Избыточное комментирование очевидного**

```rust
// ПЛОХО: очевидно даже без комментариев
fn main() {
    // начало функции main
    let x = 5; // переменная x равна 5
    // конец функции main
}

// ХОРОШО: комментируй только нетривиальные вещи

## Практический пример: документируем торгового бота

```rust
/*
    Модуль: Торговый сигнал BTC/USDT
    Стратегия: RSI + Volume Breakout
    Версия: 1.0.0
*/

/// Генерирует торговый сигнал на основе технических индикаторов.
///
/// # Arguments
/// * `price` - текущая цена актива
/// * `sma` - значение скользящей средней
/// * `rsi` - значение RSI (0-100)
/// * `volume` - текущий объём торгов
/// * `avg_volume` - средний объём за N периодов
///
/// # Returns
/// Строка с сигналом: "BUY", "SELL" или "HOLD"
fn generate_signal(
    price: f64,
    sma: f64,
    rsi: f64,
    volume: f64,
    avg_volume: f64,
) -> &'static str {
    // Объём должен подтверждать сигнал (фильтр ложных пробоев)
    let volume_ok = volume > avg_volume * 1.2; // объём на 20% выше среднего

    // Условия для покупки: цена выше SMA, RSI не перекуплен, объём есть
    let buy_signal = price > sma && rsi < 65.0 && volume_ok;

    // Условия для продажи: цена ниже SMA, RSI не перепродан, объём есть
    let sell_signal = price < sma && rsi > 35.0 && volume_ok;

    if buy_signal {
        "BUY"
    } else if sell_signal {
        "SELL"
    } else {
        "HOLD" // нет чёткого сигнала — лучше не торговать
    }
}

/// Рассчитывает уровни стоп-лосса и тейк-профита.
///
/// # Arguments
/// * `entry` - цена входа в позицию
/// * `is_long` - true для лонга, false для шорта
///
/// # Returns
/// Кортеж (stop_loss, take_profit)
fn calculate_levels(entry: f64, is_long: bool) -> (f64, f64) {
    let stop_pct = 0.04;    // стоп 4% от входа
    let profit_pct = 0.08;  // тейк 8% от входа (RR = 2.0)

    if is_long {
        // Для лонга: стоп ниже входа, тейк выше
        let stop = entry * (1.0 - stop_pct);
        let take = entry * (1.0 + profit_pct);
        (stop, take)
    } else {
        // Для шорта: стоп выше входа, тейк ниже
        let stop = entry * (1.0 + stop_pct);
        let take = entry * (1.0 - profit_pct);
        (stop, take)
    }
}

fn main() {
    // --- Рыночные данные (в реальном боте получаем от биржи) ---
    let btc_price = 43500.0;    // цена BTC/USDT
    let sma_20 = 42800.0;       // 20-часовая SMA
    let rsi_14 = 52.0;          // RSI(14) часовой
    let hour_volume = 1850.0;   // объём за последний час (BTC)
    let avg_volume_24h = 1500.0; // средний часовой объём за 24ч

    println!("=== Анализ BTC/USDT ===");
    println!("Цена: ${:.2}", btc_price);
    println!("SMA(20): ${:.2}", sma_20);
    println!("RSI(14): {:.1}", rsi_14);
    println!();

    // Генерируем сигнал
    let signal = generate_signal(btc_price, sma_20, rsi_14, hour_volume, avg_volume_24h);
    println!("Сигнал: {}", signal);

    // Если есть сигнал на покупку — рассчитываем уровни
    if signal == "BUY" {
        let (stop, take) = calculate_levels(btc_price, true);
        println!("Стоп-лосс: ${:.2}", stop);
        println!("Тейк-профит: ${:.2}", take);
        println!("Risk/Reward: {:.1}", (take - btc_price) / (btc_price - stop));
    }
}
```

## Что мы узнали

| Синтаксис | Когда использовать | Пример |
|-----------|-------------------|--------|
| `// текст` | Пояснение к строке или блоку кода | `let fee = 0.001; // комиссия Maker` |
| `/* текст */` | Большой блок, документация стратегии, временное отключение | `/* Стратегия RSI Reversal v2.1 */` |
| `/// текст` | Документация публичных функций и типов | `/// Рассчитывает PnL по сделке` |
| `# Arguments` | Секция аргументов в `///` | Описание каждого параметра |
| `# Returns` | Секция возвращаемого значения в `///` | Что возвращает функция |
| `# Example` | Секция примера в `///` | Работающий код с `assert!` |
| `# Panics` | Условия паники в `///` | Когда функция может упасть |
| `cargo doc` | Сборка HTML-документации | `cargo doc --open` |
